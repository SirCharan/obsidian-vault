### Revenue Sharing Model Discussion

- 50/50 split on funding fees and trading fees from perpetuals
- Projects can set their own trading fees (including zero)
- Minimum funding rate: 10% fixed (vs HyperLiquid’s variable 10-300%)
- No charges on swap fees or vault management

- Using open-source Charm Fi vault strategies
- Uniswap infrastructure already commoditized
- All fees collected in USDC only, no project tokens

### Time Lock Swap Features & Capabilities

- Complete DeFi stack integration:

- Perpetuals without liquidations
- Options (custom strikes, any duration down to block time)
- Everlasting/perpetual options
- Automated liquidity management via Charm Fi
- Project configuration options:

- Different buy/sell fees on Uniswap pools
- Automatic token burning mechanisms
- USDC buyback functionality
- Single-sided USDC liquidity deposits creating price floors

### Stream Swap Integration Strategy

- Challenge: Can’t charge for commoditized features (swap, vesting, pool creation)
- Potential revenue sources:

- Upfront fees for enabling perpetuals ($15-20k vs $20k standard)
- Revenue share from funding rates on funneled projects
- Optional liquidity locking vs permissionless withdrawal
- Current limitation: Projects free to withdraw liquidity (max 5% borrowed against positions)
- Marketing disadvantage: Time Lock not promoting existing DEX features

- Create onboarding documentation for projects
- Evaluate upfront fee vs revenue share models based on project confidence
- Consider dev support offering (5-15k for 2-day delivery on specific tasks)
